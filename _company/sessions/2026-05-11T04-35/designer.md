# 🎬 Motion Graphic 최종 시각 설계안 및 애니메이션 흐름 (The Engine Deep Dive)

**프로젝트 목표:** 시스템 작동 시뮬레이션(Input → Process → Output)을 오렌지색(`FF8C00`) 강조 원칙을 적용하여 권위 있고 몰입도 높은 영상으로 제작.

**핵심 비주얼 컨셉:** [The Blueprint] 및 [The Engine]의 통합, 데이터 흐름 시각화 극대화.

## 1. 디자인 시스템 확정 (Visual System Finalization)

### 1.2. 타이포그래피 스타일 (Typography Style)
*   **제목/핵심 지표:** 산세리프 계열 (예: Montserrat Bold 또는 Impact 계열). 크고 강렬하게 배치하여 임팩트 부여.
*   **설명/데이터:** 깔끔한 산세리프 (예: Roboto, Inter). 명확성과 가독성 최우선.

### 1.3. 레이아웃 및 구도 원칙 (Layout & Composition Rules)
1.  **대칭적 흐름:** 화면을 좌측(Input)에서 우측(Output)으로의 선형적인 데이터 흐름에 맞춰 구성한다.
2.  **엔진 강조:** 'Process' 단계는 가장 역동적으로 표현하며, 이 부분 전체를 `#FF8C00` 그라데이션으로 감싸 시각적 무게중심을 부여한다.
3.  **데이터 시각화:** JSON 구조와 API 결과값(`profit_path`, `system_metrics`)은 텍스트 나열이 아닌, **Flowchart 형태의 연결선과 박스**로 시각화하여 '해부(Deep Dive)' 느낌을 극대화한다.

## 2. 애니메이션 흐름 (Animation Flow / Storyboarding)

Motion Graphic은 총 4단계의 명확한 전환을 통해 시스템 작동 과정을 설명해야 합니다.

### Scene 1: Input (입력 단계)
*   **내용:** 사용자의 초기 입력(Input Data)이 시스템으로 들어오는 장면.
*   **시각화:** 화면 좌측에 'Input' 박스가 등장하며, 데이터가 화살표를 통해 중앙의 프로세스로 유입되는 모습.
*   **강조:** Input 데이터는 보조 색상(흰색/회색)으로 처리하여 시스템의 시작점을 명확히 한다.

### Scene 2: Process (처리 단계 - The Engine)
*   **내용:** 핵심 로직(API 연결, 계산, 시스템 작동)이 일어나는 과정. **가장 역동적인 부분.**
*   **시각화:** 중앙에 거대한 'Engine' 또는 'System Flowchart'가 등장한다. 이 흐름 전체는 `#FF8C00` 그라데이션으로 채워져 시각적 에너지를 폭발시킨다. 각 단계(Input → Process → Output) 간의 연결선은 오렌지색으로 강조되어 데이터의 이동을 역동적으로 보여준다.
*   **강조:** JSON 구조와 기술 명세가 이 엔진 내부에서 실시간으로 계산되는 것처럼 시각화된다.

### Scene 3: Output (결과 단계 - Profit Path)
*   **내용:** 시스템 처리 결과(Output Data)가 도출되는 장면.
*   **시각화:** 중앙의 엔진에서 최종 결과물인 'Profit Path'가 명확하게 분리되어 출력된다. 이 결과값(`profit_path`, `system_metrics`)은 가장 크고 굵은 글씨로 강조되며, 오렌지색으로 하이라이트된다.
*   **강조:** 최종 수익 경로(Profit Path)는 시각적으로 가장 매력적이고 강력하게 표현하여 시청자의 이목을 집중시킨다.

### Scene 4: Conclusion (해부 및 권위 확립)
*   **내용:** 시스템의 작동 원리 해부와 권위 강조.
*   **시각화:** 최종 결과물 위에 'Deep Dive Authority' 또는 'System Blueprint'라는 타이틀이 오렌지색으로 명확하게 제시된다. 전체 화면은 딥 그레이 배경 위에서 오렌지색 라인과 데이터로만 구성되어, 복잡한 시스템을 단순하고 권위 있게 요약한다.

## 3. 최종 제작 지침 (Production Directives)

1.  **강조 원칙 준수:** 모든 핵심 프로세스 흐름, 결과값, 그리고 'Engine'의 연결선은 **반드시 `#FF8C00` 강조 원칙을 일관되게 적용**해야 한다.
2.  **흐름 우선:** 텍스트 설명보다 시각적 흐름(Flow)이 영상의 70%를 차지하도록 설계한다.
3.  **속도감 부여:** 데이터가 처리되는 과정은 빠르고 명확하게 움직여야 하며, 정적인 정보는 최소화하고 동적인 연결에 집중한다.
