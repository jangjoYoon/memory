## ⚙️ Motion Graphic 및 Flowchart 최종 시각 설계안 (The Engine Blueprint)

Writer가 작성한 스크립트와 Business의 '시각적 권위(`FF8C00`)' 원칙을 기반으로, **'시스템 해부'** 영상에 최적화된 Motion Graphic 및 Flowchart의 최종 레이아웃과 애니메이션 흐름을 설계합니다. 이 설계안은 Developer가 즉시 구현할 수 있는 기술 요구사항(JSON 구조 및 애니메이션 시퀀스)의 기초가 됩니다.

### 1. 전체 비주얼 컨셉: 시스템 해부 (System Dissection)

*   **핵심 테마:** 혼돈(Chaos)에서 질서(Order)로의 변환, 비효율성(Bottleneck) 제거를 통한 자동화된 수익 흐름 구축.
*   **색상 팔레트 적용:**
    *   **배경/기본 구조:** 짙은 차콜 그레이 (`#1A1A1A`) 또는 미드나잇 블루 (`#0F1D3B`) - 전문성과 권위 부여.
    *   **핵심 흐름 (Automation Path):** 오렌지색 강조 원칙 (`FF8C00`) - 시스템의 핵심 작동 경로, 자동화된 부분, 그리고 수익 엔진을 시각적으로 강조.
    *   **문제점/병목 지점:** 강렬한 레드 (`#E53935`) - 비효율성, 손실, 정체 구간을 명확히 표시.
    *   **보조 요소:** 화이트/라이트 그레이 - 텍스트 및 보조 정보의 가독성 확보.

### 2. Flowchart 레이아웃 설계 (The Blueprint)

Flowchart는 전체 영상의 **'엔진 해부'** 구조를 반영하여, 세 개의 핵심 엔진이 상호작용하는 모듈화된 다이어그램으로 구성합니다.

*   **레이아웃:** 중앙 집중형(Hub-and-Spoke) 구조를 채택하며, 세 가지 엔진을 중심으로 데이터 흐름이 교차하도록 설계합니다.
*   **모듈 분리 (Developer 요구사항):** 각 엔진은 독립적인 모듈로 분리되어야 하며, JSON 데이터 구조에 따라 동적으로 연결됩니다.

| 모듈 | 명칭 | 시각적 역할 | 강조 색상 |
| :--- | :--- | :--- | :--- |
| **Engine 1** | 입력 엔진 (Input Engine) | 데이터 수집 및 정제 과정 | 오렌지 (`FF8C00`) |
| **Engine 2** | 처리 엔진 (Processing Engine) | 시스템의 핵심 로직 및 자동화 알고리즘 | 오렌지 (`FF8C00`) |
| **Engine 3** | 출력 엔진 (Output Engine) | 수익화 및 결과물 도출 과정 | 오렌지 (`FF8C00`) |
| **Bottleneck** | 병목 지점 (Bottleneck Points) | 비효율적인 수동 작업 또는 데이터 누락 구간 | 레드 (`#E53935`) |

### 3. 애니메이션 흐름 설계 (Motion Flow Sequence)

애니메이션은 **'문제 제기 → 해부 → 해결책 제시'**의 논리적 흐름을 따라 역동적으로 진행되어야 합니다.

| 시간대 (스크립트 연계) | 시각적 액션 (Motion) | 오디오/텍스트 동기화 | 강조 효과 |
| :--- | :--- | :--- | :--- |
| **0:00 - 0:25 (Hook)** | 화면 전체에 복잡한 데이터 흐름이 빠르게 스캔됨. 갑자기 중앙의 '엔진' 아이콘들이 분리되며 정렬되기 시작함. | 강렬한 후크 문구와 함께 에너지 넘치는 사운드 효과. | **FF8C00**으로 엔진 경계선이 강조됨. |
| **0:25 - 1:30 (문제 정의)** | 비효율적인 수동 작업(빨간색 박스)들이 화면을 가로지르며, 이들이 시스템의 흐름을 막는 '벽'처럼 표현됨. | "자동화 병목 지점" 언급 시, 빨간색으로 표시된 구간에 집중적으로 줌인. | **#E53935**를 사용하여 문제점을 명확히 부각. |
| **1:30 - 4:30 (엔진 해부)** | **[Engine 1]** 데이터가 입력되어 정제되는 과정이 오렌지색 흐름으로 매끄럽게 연결됨. 병목 지점(레드)을 통과하는 과정을 시뮬레이션. | 각 엔진별 설명에 맞춰 해당 모듈이 활성화되며 작동하는 모습을 보여줌. | **FF8C00**의 에너지 흐름이 가장 역동적으로 움직임. |
| **4:30 - End (결론)** | 세 엔진이 완벽하게 동기화되어 하나의 거대한 '수익 엔진'으로 통합되는 최종 시퀀스. 모든 병목 지점(레드)이 사라지고 오렌지색 흐름만 남음. | "핵심 엔진 3가지" 요약과 함께, 시스템의 완성된 모습을 보여줌. | **FF8C00**이 전체 화면을 감싸며 권위적인 마무리 연출. |

### 4. 디자인 가이드라인 (Design Guidelines)

*   **타이포그래피:** 제목 및 핵심 수치(예: $497, 3가지 엔진)에는 산세리프 계열의 두꺼운 폰트(예: Montserrat Bold 또는 Impact 스타일)를 사용하여 시각적 권위를 확보합니다.
*   **흐름선 (Flow Lines):** 모든 데이터 흐름과 에너지 이동은 **오렌지색(`FF8C00`)**의 선형 애니메이션으로 표현하며, 이는 시스템의 '에너지'와 '수익 경로'를 상징합니다.
*   **모듈 분리:** 각 엔진 모듈 사이에는 명확한 간격(Whitespace)을 두어 복잡함을 줄이고, 시청자가 한 번에 하나의 개념에 집중하도록 유도합니다.
