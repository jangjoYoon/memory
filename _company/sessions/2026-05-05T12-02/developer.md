### 1. 핵심 시스템 흐름 정의 (Input → Process → Output)

이 섹션은 영상의 핵심적인 논리 구조이며, Motion Graphic과 Flowchart의 기본 골격이 됩니다. 모든 단계는 FF8C00 색상으로 강조되어야 합니다.

| 단계 | 명칭 | 기술적 설명 (Developer 관점) | 시각적 요구사항 (Designer 지시 사항) |
| :--- | :--- | :--- | :--- |
| **Input** | 데이터 유입 및 긴장감 극대화 | 외부 데이터(예: 사용자 입력, API 호출)가 시스템으로 들어오는 단계. 잠재적 오류나 정보의 불확실성을 강조해야 함. | 배경은 어둡게 처리하고, 데이터가 들어오는 지점에 FF8C00으로 깜박이는 경고 표시(Error/Warning)를 삽입하여 긴장감을 극대화. |
| **Process** | 시스템 설계 및 엔진 해부 | 핵심 알고리즘, 데이터 처리 로직, 수익 경로 계산이 이루어지는 단계. '시스템 설계자'의 권위를 시각적으로 입증해야 함. | 복잡한 흐름을 명확하게 분할하고, 각 모듈(Input/Process) 간의 연결선에 FF8C00으로 동기화된 화살표를 사용하여 데이터 흐름을 명확히 시각화. |
| **Output** | 결과 도출 및 수익 경로 시각화 | 최종 결과물(Profit Path)이 도출되고, 이것이 어떻게 구독자 증가나 수익으로 연결되는지를 보여주는 단계. | 가장 밝고 강력한 오렌지색(FF8C00)을 사용하여 최종 결과값과 수익 경로를 강조. 'The Blueprint'의 핵심 가치(수익 엔진)가 명확하게 드러나도록 시각적 권위를 확보. |

### 2. Flowchart 상세 구조 및 기술 흐름 (Developer 산출물 기반)

Motion Graphic 제작에 필요한 구체적인 데이터 포인트와 연결 로직을 정의합니다. 이는 Motion Graphic의 애니메이션 순서를 결정하는 데 사용됩니다.

**[Flowchart 핵심 노드 및 연결]**

1.  **Start Node:** 시스템 초기화 (Input Data Load)
    *   *연결:* Input 단계로 이동.
2.  **Decision Node:** 데이터 유효성 검사 (Validation Check)
    *   *조건:* 데이터 오류 발생 시 (FF8C00 경고 표시 활성화).
    *   *분기:* Valid $\rightarrow$ Process, Invalid $\rightarrow$ Error Handling Loop.
3.  **Process Block 1:** 핵심 로직 실행 (Profit Path Calculation Engine)
    *   *내부 흐름:* Input 데이터 $\rightarrow$ 알고리즘 적용 $\rightarrow$ 중간 결과값 산출.
4.  **Process Block 2:** 결과 포맷팅 및 시각화 (Output Formatting)
    *   *연결:* Process Block 1의 최종 결과값을 Output 단계로 전달.
5.  **Output Node:** 최종 결과물 제시 (Revenue/Subscription Path Visualization)
    *   *강조:* 이 노드 전체를 FF8C00으로 채우고, 수익 경로(Profit Path)가 시각적으로 명확하게 연결되도록 디자인.

### 3. Motion Graphic 및 Flowchart 애니메이션 요구사항 (FF8C00 적용 원칙)

Motion Graphic 제작 시 다음의 시각적 규칙을 엄격히 준수하여 '시각적 권위'를 확보해야 합니다.

1.  **색상 사용:** 모든 핵심 흐름, 경고, 최종 결과는 **오렌지색 (FF8C00)**으로 처리한다. 배경 및 보조 정보는 중립적인 색상을 유지하여 오렌지색의 강조 효과를 극대화한다.
2.  **흐름 동기화:** Input $\rightarrow$ Process $\rightarrow$ Output의 흐름은 부드럽지만 명확하게 연결되어야 하며, 각 단계가 다음 단계로 넘어갈 때 FF8C00으로 빛나는 애니메이션 효과를 적용하여 시스템의 작동을 시각적으로 증명한다.
3.  **긴장감 표현:** Input 단계에서는 데이터 유입 시 잠시 멈춤(Pause) 또는 오류 아이콘이 나타나며, Process 단계에서 복잡한 계산이 이루어질 때 역동적인 움직임을 사용하여 '엔진 해부'의 긴장감을 전달한다.
4.  **최종 강조:** Output 단계에서는 모든 흐름이 성공적으로 완료되었음을 FF8C00으로 강력하게 선언하며, 최종 수익 경로(Profit Path)가 가장 눈에 띄도록 디자인한다.
