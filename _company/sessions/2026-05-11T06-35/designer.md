# 🎬 Motion Flow 최종 시각 설계안 (The Engine Blueprint)

**목표:** 시스템 작동 시뮬레이션 로직을 오렌지색(`FF8C00`) 강조 원칙을 통해 단순화하고 권위를 극대화하여, 복잡한 흐름을 명확하게 전달한다.

**핵심 원칙:**
1. **강조 색상:** 오렌지색 (`FF8C00`)은 시스템의 핵심적인 'Profit Path' 또는 'Engine' 부분을 시각적으로 부각시키는 데 사용된다.
2. **흐름:** Input → Process → Output의 논리적 흐름을 명확하게 시각화한다.
3. **스타일:** 대담하고 시각적으로 충격적인 미스터 비스트 스타일을 유지하여 권위를 강화한다.

## 1. 전체 시각 흐름 (Flowchart) 개요

| 단계 | 설명 | 시각적 강조 요소 | 예상 길이/시간 배분 |
| :--- | :--- | :--- | :--- |
| **Phase 1: Input (입력)** | 사용자의 초기 데이터 및 목표 설정 단계. | 일반적인 배경색, 명확한 입력 박스. | 10% |
| **Phase 2: Process (처리 엔진)** | 시스템 로직(API Mock-up)이 작동하며 데이터가 처리되는 핵심 과정. | **오렌지색 (`FF8C00`)**으로 강조된 중앙 프로세스 블록 및 연결선. 복잡한 흐름을 단순화하여 시각적 임팩트 극대화. | 60% |
| **Phase 3: Output (결과)** | 최종 결과물(Profit Path)이 도출되는 단계. | 오렌지색으로 강조된 최종 결과 지표 및 성공 경로. 명확하고 강력한 마무리. | 30% |

## 2. 상세 시각 요소 및 컬러 적용

### A. 컬러 팔레트 정의
*   **Primary Accent (강조):** `#FF8C00` (오렌지색) - 시스템의 핵심 동력, 수익 경로(Profit Path), 주요 전환점 강조.
*   **Secondary Color:** 짙은 회색/검정 (`#1A1A1A`) - 배경 및 텍스트로 권위와 전문성 부여.
*   **Neutral Color:** 밝은 회색 (`#F5F5F5`) - 입력/출력 영역 구분 및 여백 확보.

### B. Motion Flow 상세 구성 (Blueprint)

1.  **시작점 (Input):** 화면 좌측 상단에 사용자의 목표(Goal)가 명확히 정의되는 박스에서 시작한다.
2.  **데이터 흐름 (The Engine):**
    *   Input 데이터가 중앙의 **Process Block**으로 유입된다. 이 Process Block 전체와 그 내부의 연결선은 **`#FF8C00`**으로 채워져, 시스템이 '작동 중'임을 시각적으로 강력하게 전달한다.
    *   복잡한 API 로직이나 데이터 흐름은 단순화된 화살표와 명확한 레이블링을 통해 직관성을 확보한다. (기술적 정확성 유지)
3.  **결과 도출 (Output):**
    *   Process Block의 최종 결과가 **Profit Path**라는 이름으로 시각적으로 분리되어 강조된다. 이 Profit Path 라인은 오렌지색으로 처리되어, 이것이 시스템의 궁극적인 목표임을 명시한다.
4.  **마무리:** 전체 흐름은 역동적이고 대담한 움직임(Motion)을 사용하여, 단순한 정보 전달이 아닌 '시스템 해부'의 느낌을 극대화한다.
