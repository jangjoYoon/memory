| :--- | :--- | :--- | :--- |
| **Raw Data Inflow** | 수많은 데이터 조각(텍스트, 그래프, 코드 스니펫 등)이 무질서하게 화면을 가득 채우며 중앙 시스템으로 쏟아져 들어오는 애니메이션. | 데이터의 비정형성(Unstructured Data)과 초기 노이즈(Noise)를 시각화. API 호출 실패나 데이터 정제 필요성을 암시하는 오류 메시지 삽입. | 기본 회색/검은색 |
| **Market Needs** | 시장 니즈(Pain Points)가 명확한 아이콘이나 텍스트 박스로 분리되어 입력되는 모습. | 시장의 요구사항이 어떻게 시스템 설계의 초기 제약 조건(Constraint)으로 작용하는지 시각화. | 오렌지 (`FF8C00`) |
| **Initial Resources** | 초기 자원(시간, 예산, 코드베이스 등)이 시스템에 연결되는 모습. | 사용 가능한 리소스의 한계와 잠재적 병목 현상(Bottleneck)을 암시하는 시각적 표현. | 오렌지 (`FF8C00`) |

### 2. Phase 2: Process (시스템 설계 - The Engine) - 핵심 기술 흐름 및 정제 과정

**목표:** Input된 혼란스러운 데이터를 체계적인 'Input → Process → Output'의 논리적 엔진으로 전환하는 과정을 시각적으로 명확히 하여, 개발자의 시스템 설계 능력(System Design Authority)을 입증합니다. **이 단계가 가장 오렌지색 강조 원칙(`FF8C00`)을 집중 적용해야 합니다.**

| 요소 | 시각적 요구사항 (B-roll/Motion Graphic) | 기술적 강조점 (Developer Focus) | 강조 색상 |
| :--- | :--- | :--- | :--- |
| **Input → Process** | Raw Data가 중앙 처리 장치(CPU/Engine)로 유입되며, 복잡한 알고리즘과 로직이 데이터 흐름을 따라 실시간으로 적용되는 모습. | **핵심:** 데이터 정제(Data Cleansing), 변환(Transformation), 검증(Validation) 모듈의 작동 과정을 명확히 보여주기. (예: ETL 파이프라인, 데이터 모델링). | **오렌지 (`FF8C00`)** |
| **The Engine Core** | Input과 Process가 복잡하게 얽힌 기어 또는 연결된 노드(Node) 구조를 중심으로, 오렌지색 선들이 논리적 흐름을 따라 움직이는 모습. | 시스템 아키텍처(Architecture)의 계층 구조와 모듈 간의 의존성(Dependency) 관계를 시각적으로 해부. (P1 모듈의 기술적 흐름 강조). | **오렌지 (`FF8C00`)** |
| **Optimization Loop** | 프로세스 단계에서 발생하는 반복적인 피드백 루프(Feedback Loop)와 최적화 과정이 끊임없이 순환하며 결과물의 품질을 높이는 모습. | 자동화 스크립트의 반복 실행, 에러 처리 로직(Error Handling), 그리고 성능 최적화(Performance Optimization) 알고리즘의 작동 시퀀스. | 오렌지 (`FF8C00`) |

### 3. Phase 3: Profit Path Link (수익 경로 분기) - 결과 도출 및 행동 유도

**목표:** 정제된 시스템 설계가 구체적인 수익화 경로(Profit Path)로 명확하게 연결되는 지점을 시각적으로 강조하여, 기술적 실행이 곧 비즈니스 성과임을 입증합니다.

| 요소 | 시각적 요구사항 (B-roll/Motion Graphic) | 기술적 강조점 (Developer Focus) | 강조 색상 |
| :--- | :--- | :--- | :--- |
| **Path Divergence** | 중앙 엔진에서 명확하게 분기되는 여러 개의 수익 경로(Path A, Path B 등). 각 경로는 구체적인 상품이나 서비스로 연결됨. | 각 수익 경로가 어떤 기술적 모듈(Input/Process)에 의해 생성되었는지 시각적으로 매핑. (예: 데이터 정제 결과가 리드 마그넷으로 전환되는 흐름). | **오렌지 (`FF8C00`)** |
| **Profit Path Visualization** | 각 경로가 오렌지색 선을 따라 명확하게 분리되어, 최종 목표(구독자 1만 도달)로 수렴하는 시각적 흐름. | 수익화 전략이 단순한 추측이 아닌, 설계된 시스템의 논리적 결과임을 강조. (수익 경로 = 시스템 설계). | **오렌지 (`FF8C00`)** |
| **Call to Action Link** | 최종 결과물(Profit Path)에서 시청자가 취해야 할 다음 행동(다운로드 버튼 등)으로 연결되는 명확한 화살표. | 기술적 흐름이 실제 사용자 행동(Conversion)으로 이어지는 직접적인 인과관계를 강조. | 오렌지 (`FF8C00`) |
