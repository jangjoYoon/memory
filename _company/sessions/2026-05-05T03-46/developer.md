### 1. 핵심 시스템 모듈 상세 분해 (Input $\rightarrow$ Process $\rightarrow$ Output)

영상에서 가장 많은 시간과 권위를 할애해야 할 부분입니다. 각 단계별로 시각화할 구체적인 데이터와 기술적 흐름을 명확히 정의합니다.

| 단계 | 기술적 설명 (Developer View) | 영상 내 핵심 메시지 (Audience View) | 필수 시각 자료/데이터 포인트 | 강조 색상 |
| :--- | :--- | :--- | :--- | :--- |
| **Input (입력)** | 외부 데이터 수집 및 정제 (API Call, Data Ingestion) | "원료 확보: 시스템의 시작점" | 실제 데이터 소스(DB/API 화면), 데이터 노이즈 제거 과정 시각화. | $\text{FF8C00}$ |
| **Process (처리)** | AI 기반 알고리즘 적용 및 변환 (Core Logic, Transformation) | "엔진 해부: 시스템의 핵심 로직" | 복잡한 알고리즘 흐름도(Flowchart), 데이터가 처리되는 내부 프로세스 시각화. | $\text{FF8C00}$ |

### 2. 편집을 위한 구체적인 기술 흐름 및 데이터 포인트 보강

영상 스크립트와 B-roll 구성에 직접적으로 사용될 수 있도록, 각 단계에서 강조해야 할 **구체적인 기술 용어**와 **데이터 연결 지점**을 명시합니다.

*   **Input 시각화 강화:**
    *   **포인트:** 데이터의 '질(Quality)'이 최종 결과에 미치는 영향을 강조해야 합니다.
    *   **요소:** API 호출 성공/실패 상태, 데이터 정제(Cleaning) 단계에서 발생하는 오류 처리 로직을 짧게 삽입하여 시스템의 견고함을 보여줍니다. (기술적 권위 확보)
*   **Process 시각화 강화 (핵심):**
    *   **포인트:** '시스템 설계자'로서의 권위를 극대화하기 위해, 단순한 흐름이 아닌 **알고리즘의 논리적 구조**를 강조해야 합니다.
    *   **요소:** Input 데이터가 Process 단계에서 어떻게 변환되어 Output으로 이어지는지, 각 모듈(Module) 간의 의존성(Dependency)을 화살표와 레이블로 명확히 표시합니다. (Profit Path와의 연결 지점 명시)
*   **Output 시각화 강화 (수익 경로):**
    *   **포인트:** 수익 경로가 단순한 결과물이 아니라, **시스템 설계의 직접적인 결과**임을 강조해야 합니다.
    *   **요소:** 최종 Output 단계에서 리드 마그넷 다운로드 버튼이나 유료 상품으로 연결되는 시각적 흐름을 가장 오렌지색(`FF8C00`)으로 강조하여 행동 유도를 극대화합니다.

### 3. Designer 산출물 통합 지침 (Visual Execution Guide)

Designer가 제작할 최종 비주얼에 대한 핵심 지침입니다.

*   **핵심 비주얼 원칙:** 모든 시각적 요소는 오렌지색 강조 원칙(`FF8C00`)을 기반으로 하며, 시스템의 복잡성을 **명확한 계층 구조(Hierarchy)**로 단순화하여 보여줍니다.
*   **B-roll 구성 지침:**
    1.  **Hook (문제 정의):** 시청자의 고통스러운 상황(Input)을 극적으로 보여주는 클로즈업 샷으로 시작합니다.
    2.  **Engine Breakdown (Process):** 복잡한 데이터 흐름 다이어그램과 함께, 실제 코드가 실행되는 듯한 모션 그래픽(Motion Graphic)을 삽입하여 기술적 깊이를 시각화합니다.
    3.  **Profit Path (결과):** 최종 수익 그래프와 다운로드 버튼을 오렌지색으로 강조하며 제시하여, 시스템이 제공하는 **명확한 가치**를 즉시 인지하게 합니다.
