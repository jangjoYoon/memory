# Motion Graphic 최종 가이드라인 (FF8C00 적용)

## 1. 시각 시스템 정의
*   **주요 색상:** `FF8C00` (오렌지 강조 원칙)
*   **스타일:** 대담하고 시각적으로 충격적인 미스터 비스트 스타일 유지.
*   **폰트:** 제목은 고딕 계열의 강렬한 폰트, 본문은 가독성이 높은 산세리프 계열 사용.

## 2. 시스템 흐름 Flowchart (Blueprint)
다음은 Input → Process → Output의 동적 시뮬레이션 흐름을 나타내는 최종 편집 가이드라인입니다.

**[Scene 1: Input - 데이터 유입]**
*   **비주얼:** 복잡한 데이터가 화면에 빠르게 유입되는 모습.
*   **강조:** 입력되는 데이터 라인 전체에 `FF8C00` 강조 효과 적용.
*   **내레이션:** "시스템으로 들어오는 원료(Input)를 확인합니다."

**[Scene 2: Process - 엔진 해부 (핵심 시각화)]**
*   **비주얼:** 데이터가 복잡한 알고리즘과 연결되어 처리되는 동적 시뮬레이션.
*   **강조:** 프로세스 단계(알고리즘 실행)의 연결선과 핵심 연산 부분에 `FF8C00`을 집중적으로 적용하여 에너지 흐름을 명확히 표현. (Developer 검증 완료된 로직 반영)
*   **내레이션:** "AI 기반 엔진이 데이터를 분석하고 최적화하는 과정입니다."

**[Scene 3: Output - 결과 도출]**
*   **비주얼:** 정제되고 명확한 최종 결과물(Profit Path)이 화면에 나타남.
*   **강조:** 최종 결과물(`Output`)은 가장 밝고 강력한 `FF8C00`으로 강조하여 시청자의 시선을 집중시킵니다.
*   **내레이션:** "최적화된 수익 경로(Profit Path)가 도출되었습니다."

## 3. 편집 지침 (Synchronization Check)
1.  **기술 정합성:** 모든 시각적 흐름은 Developer가 검증한 시스템 로직과 완벽하게 동기화되어야 합니다.
2.  **색상 일관성:** `FF8C00`은 단순한 강조가 아닌, 시스템의 '에너지'와 '핵심 가치'를 나타내는 핵심 요소로 사용됩니다.
3.  **속도 조절:** Input 단계는 빠르고 혼란스럽게, Process 단계는 정교하고 집중되게, Output 단계는 명확하고 강력하게 속도를 조절하여 시청자의 몰입을 유도합니다.
