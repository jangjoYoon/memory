# 유튜브 영상 최종 시각적 흐름 및 편집 가이드라인 (The Blueprint 기반)

본 가이드라인은 '시안 A: The Blueprint'를 기반으로, 브랜드의 핵심 원칙인 **오렌지색 강조 원칙**을 철저히 반영하여 시청자의 몰입과 시스템 이해도를 극대화하는 최종 영상 편집 흐름(Flowchart)을 정의합니다.

## 1. 비주얼 시스템 요약 (Brand Visual System Summary)

| 요소 | 설정 값 | 적용 원칙 |
| :--- | :--- | :--- |
| **핵심 컬러** | 오렌지 (Profit Path 강조) | 수익 지점, 시스템의 흐름, 행동 유도에만 사용. |
| **보조 컬러** | 블랙/화이트/그레이 | 배경 및 텍스트 가독성 확보를 위한 여백(Whitespace) 극대화. |
| **타이포그래피** | 구조 강조형 (Bold & Sans-serif) | 시스템의 논리적 흐름을 명확히 전달. |
| **핵심 컨셉** | The Profit Path (수익 경로 시각화) | Input → Process → Output의 명확한 연결고리 제시. |

## 2. 영상 시각적 흐름 (Flowchart & Editing Guide)

영상은 전체적으로 **좌측에서 우측으로, 위에서 아래로** 흐르는 논리적인 시스템 구축 과정을 시각적으로 증명해야 합니다. 오렌지색은 이 흐름의 '수익 지점'에만 집중적으로 사용되어야 합니다.

### 🎬 Scene Breakdown & Visual Execution

| 시간대 (예시) | 영상 내용 (스크립트 연계) | 시각적 요소 (Visuals) | 오렌지 강조 적용 (Actionable) |
| :--- | :--- | :--- | :--- |
| **0:00 - 0:15** | **후크 & 문제 제기:** "당신의 수익은 어디서 오는가? 시스템 설계자가 되는 법" | 강렬한 타이포그래피 등장. 복잡하고 혼란스러운 초기 상태의 이미지 (Before). | **[강조]** '수익'이라는 단어에 오렌지색 하이라이트 적용. |
| **0:15 - 0:45** | **시스템 정의:** 수익화 시스템 설계자의 역할 소개 및 필요성 강조. | '시스템 설계자(Architect)' 키워드 등장. 복잡한 개념을 단순화하는 다이어그램 시작. | **[강조]** 시스템의 전체 구조를 감싸는 오렌지색 테두리 또는 배경 블록으로 정의. |

## 3. 최종 편집 가이드라인 (Final Editing Directives)

1.  **Flowchart 스타일:** 모든 시스템 흐름(Input, Process, Output)은 **오렌지색 선과 박스**를 사용하여 시각적으로 연결되어야 합니다. 일반적인 파란색/회색의 논리적 흐름이 아닌, '수익 경로'로서의 흐름을 강조합니다.
2.  **오렌지 사용 규칙:** 오렌지색은 **절대 과도하게 사용하지 않습니다.** 오직 다음 세 가지 지점에만 집중적으로 적용합니다:
    *   시스템의 시작점 (Input).
    *   핵심 작동 단계 (Process) 전체.
    *   최종 결과 및 행동 유도 (Output & CTA).
3.  **가독성 확보:** 배경은 무조건 **화이트 또는 짙은 그레이(#1A1A1A)**를 사용하여 오렌지색이 튀어나오도록 대비를 극대화합니다. 모든 텍스트는 명확한 Sans-serif 계열(예: Inter, Montserrat)을 사용하며, 여백(Whitespace)을 충분히 확보하여 시각적 피로도를 낮춥니다.
4.  **템포 조절:** 시스템 설명 부분(Process)에서는 속도를 약간 늦추어 시청자가 각 단계의 논리를 이해하도록 시간을 할애하고, 최종 결과 및 CTA 부분에서 속도를 높여 행동을 촉구합니다.
