## 🎬 Input 단계 Motion Graphic 및 Flowchart 최종 초안 (긴장감 극대화 버전)

본 산출물은 영상의 **Input 단계**에서 시청자의 긴장감을 극대화하고 데이터 유입 시 발생할 수 있는 오류와 시스템적 위험을 강조하기 위해 설계되었습니다. 오렌지색(`FF8C00`) 강조 원칙을 최우선으로 적용합니다.

### 1. 핵심 Motion Graphic 시퀀스 상세 요구사항 (Input Phase)

**목표:** 데이터가 시스템에 유입되는 과정이 단순한 입력이 아닌, **보안 및 안정성 검증**의 긴장된 과정임을 시각적으로 전달합니다.

| 시퀀스 # | 시간 배분 (예상) | 시각적 요소 (Visual Element) | 오렌지 강조 (`FF8C00`) 적용 | 사운드/효과 (SFX/Motion) |
| :---: | :---: | :--- | :--- | :--- |
| **1. 시스템 초기화** | 0:00 - 0:03 | 화면 전체가 어두운 배경(`#1A1A1A`) 위에 미세한 회로 기판(Circuit Board) 패턴이 천천히 나타남. 중앙에 'INPUT' 타이틀 등장. | 타이틀 및 초기 로딩 바는 오렌지색으로 깜빡임. | 낮은 톤의 시스템 부팅음, 미세한 전기 잡음(Static). |
| **2. 데이터 스트림 유입** | 0:03 - 0:08 | 여러 개의 데이터 스트림(파이프라인)이 화면 외부에서 폭발적으로 유입됨. 이 스트림들은 복잡하고 빠르게 움직여야 함. | 모든 데이터 스트림의 경계선과 흐름 라인은 **오렌지색(`FF8C00`)**으로 명확하게 표시. | 고주파의 데이터 흐름 소리, 빠른 틱톡(Ticking) 사운드. |
| **3. 오류 감지 (긴장 최고조)** | 0:08 - 0:15 | 유입된 데이터 스트림 중 일부가 불안정해지며 빨간색/오렌지색 경고 표시(`ERROR` 또는 `WARNING`)를 발생시킴. 데이터 흐름이 잠시 정지하거나 왜곡됨. | 오류 메시지 박스, 경고 플래그는 **강렬한 오렌지색**으로 깜빡이며 화면을 압도함. | 날카로운 경고음(Alarm), 시스템 오류 톤의 노이즈 증폭. |
| **4. 데이터 정제 (Process 시작)** | 0:15 - 0:20 | 불안정한 데이터가 중앙의 'PROCESS' 모듈로 빨려 들어가며, 내부에서 필터링 및 정제되는 시각적 효과(예: 입자가 정리됨). | 프로세스 모듈 내부의 처리 과정은 **오렌지색**으로 빛나며 역동적으로 움직임. | 데이터 정제 시 발생하는 미세한 계산음, 부드러운 흐름 소리 전환. |

### 2. Flowchart 상세 설계 (Input → Process)

**목표:** 기술적 흐름을 오렌지색 강조 원칙에 맞춰 명확하게 해부하여 시각적 권위를 확보합니다.

#### A. Input 단계 Flowchart (긴장감 및 오류 강조)

*   **형태:** 다이아몬드(Decision)와 사각형(Process)을 혼합한 형태를 사용하며, 모든 연결선은 **오렌지색(`FF8C00`)**으로 처리합니다.
*   **노드 구성:**
    1.  **Start (Input Source):** 외부 데이터가 들어오는 지점. (배경: 어두운 회로판)
    2.  **Data Ingestion:** 데이터 수집 단계. (오렌지색 박스)
    3.  **Validation Check (Decision):** 데이터 유효성 검사. (다이아몬드 형태, 'Valid?' / 'Error?')
        *   **Path 1 (Valid):** 다음 단계로 진행 (오렌지색 화살표).
        *   **Path 2 (Error):** 오류 발생 시, **빨간색 경고 박스**로 분기. (오렌지색 화살표)
    4.  **Error Handling:** 오류 발생 시, 데이터 스트림이 차단되고 'Rejection' 상태로 표시됨. (강렬한 오렌지색 강조)
    5.  **Output to Process:** 정제된 데이터가 프로세스로 전달되는 지점.

#### B. Process 단계 Flowchart (시스템 해부 및 권위 확보)

*   **형태:** 순차적이고 논리적인 흐름을 강조하는 선형 다이어그램(Linear Flow)을 사용합니다.
*   **노드 구성:**
    1.  **Process Module A (Normalization):** 데이터 정규화 단계. (오렌지색 박스)
    2.  **Process Module B (Engine Calculation):** 핵심 알고리즘 적용 단계. (오렌지색 박스, 가장 큰 영역으로 강조)
    3.  **Process Module C (Output Formatting):** 결과물 포맷팅 단계. (오렌지색 박스)
*   **연결 원칙:** 각 모듈 간의 연결은 **'시스템적 의존성'**을 나타내며, 오렌지색 화살표로 명확하게 표시하여 데이터가 어떻게 변환되는지를 시각적으로 보여줍니다.

### 3. 최종 기술적 지침 요약 (Designer 전달용)

1.  **컬러 팔레트 고정:** 모든 시스템 흐름, 핵심 데이터, 경고는 **오렌지색 (`#FF8C00`)**으로 통일합니다. 배경은 차콜 그레이/블랙을 유지하여 대비를 극대화합니다.
2.  **긴장감 설계:** Input 단계에서는 '데이터 유입'과 '오류 감지' 시퀀스에 가장 많은 시간(약 30%)을 할애하여 긴장감을 조성해야 합니다.
3.  **Motion Focus:** 데이터 스트림의 움직임은 빠르고 역동적이어야 하며, 오류 발생 시에는 일시적으로 정지하거나 왜곡되는 **디지털 글리치(Glitch) 효과**를 적용합니다.
4.  **권위 확보:** Process 단계에서는 복잡한 알고리즘이 명확하게 분리되어 해부되는 느낌을 주어 '시스템 설계자'로서의 권위를 시각적으로 뒷받침해야 합니다.
