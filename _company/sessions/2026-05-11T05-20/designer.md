## 1. 디자인 원칙 요약
*   **핵심 강조 색상:** 오렌지색 (`FF8C00`) - 시스템의 핵심 경로, 수익화 지표(Profit Path), 주요 전환점을 시각적으로 부각.
*   **스타일:** 대담하고 시각적으로 충격적인 미스터 비스트 스타일 (권위 극대화).
*   **흐름:** Input → Process → Output의 명확한 흐름을 오렌지색 화살표로 연결하여 시스템 해부(Deep Dive)의 권위를 강화.

## 2. JSON 구조 기반 시각 요소 매핑
| JSON/데이터 필드 | 시각적 표현 (Visual Element) | 강조 색상 적용 | 애니메이션 역할 |
| :--- | :--- | :--- | :--- |
| **Input Data** | 시작점, 원천 데이터 블록 | 기본 회색/흰색 배경 | 화면 진입 및 초기 설정 |
| **Process Logic** | 시스템 엔진 내부의 복잡한 흐름 (노드 연결) | **오렌지색 (`FF8C00`)** 화살표와 노드 간 연결선 | 데이터 처리 과정 시각화, 핵심 로직 강조 |
| **Profit Path** | 최종 수익 경로 지표 | **오렌지색 (`FF8C00`)**으로 채워진 선 또는 게이지 | 목표 달성 및 결과 강조 (가장 중요한 출력) |
| **Output Result** | 최종 시각화된 시스템 결과물 | 오렌지색 테두리/배경 | 최종 결과물의 명확한 제시 |

## 3. 애니메이션 흐름 (Storyboarding Flowchart)
**[Scene 1: Input & Setup]**
*   **시작:** 화면 중앙에 'Input Data' 블록이 등장한다. (기본 색상)
*   **액션:** 데이터가 시스템으로 유입되는 과정(화살표)을 보여준다.

**[Scene 2: Engine Deep Dive - Process Visualization]**
*   **전환:** Input에서 Process로 넘어가는 지점에서 **오렌지색 (`FF8C00`)**의 강력한 흐름선이 등장한다. (엔진 해부 시작)
*   **흐름:** 복잡한 JSON 구조(노드와 연결선)가 역동적으로 움직이며 데이터 처리 과정을 시각화한다. 모든 주요 로직 노드는 오렌지색으로 하이라이트된다.

**[Scene 3: Profit Path Visualization]**
*   **핵심 강조:** Process의 결과물에서 최종 목표인 'Profit Path' 지표가 명확하게 분리되어 나타난다.
*   **시각화:** 이 경로 전체는 **오렌지색 (`FF8C00`)**으로 채워져, 시스템이 어떻게 수익 흐름을 설계했는지 직관적으로 보여준다.

**[Scene 4: Final Output & Authority]**
*   **결론:** 최종 결과물(Output)이 화면에 제시된다.
*   **마무리:** 전체 시각적 흐름은 오렌지색 강조 원칙(`FF8C00`)을 통해 강력하게 마무리되며, 채널의 권위와 시스템의 완벽함을 시각적으로 각인시킨다.

## 4. 기술 및 스타일 지침 (Developer/Motion Graphic 참고용)
*   **Color Palette:** Primary: `#FF8C00` (오렌지), Secondary: `#1A1A1A` (짙은 회색 배경), Accent: `#FFFFFF` (텍스트).
*   **Motion Style:** 빠르고 날카로운 전환(Sharp Transitions), 데이터 흐름을 강조하는 역동적인 모션 그래픽(Flowchart 스타일).
*   **Emphasis Rule:** 모든 핵심 로직 연결, 수익 경로(`Profit Path`), 그리고 시스템의 핵심 엔진은 **반드시 오렌지색 (`FF8C00`)으로 처리하여 시청자의 시선을 즉시 집중시킨다.**
