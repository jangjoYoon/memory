### 2. 핵심 엔진 해부: 시스템의 3단계 흐름 (Input → Process → Output)

**(화면 전환: 화면을 세 개의 명확한 박스로 분할하며, 각 단계에 오렌지색(`FF8C00`) 테두리를 적용)**

**내레이션:** "우리가 구축할 이 수익 엔진은 단순하지 않습니다. 이것은 **Input(투입) → Process(처리) → Output(산출)**의 명확한 3단계 흐름으로 이루어져 있습니다. 시스템 설계자는 이 세 단계를 완벽하게 이해해야 합니다."

#### A. Input (투입 단계): 자원 및 데이터 정의
**(화면: 'Input' 박스 강조. 여기에 필요한 원자재/데이터가 들어오는 시각적 표현)**

**내레이션:** "첫 번째 단계는 **Input**, 즉 시스템에 투입될 자원과 데이터를 명확히 정의하는 것입니다. 이것이 당신의 수익 경로를 결정하는 가장 기초적인 재료입니다."

*   **핵심 데이터 포인트 (Data Points):**
    *   **시장 데이터 (Market Data):** 잠재 고객의 니즈, 경쟁사 가격 정보 등 외부에서 수집할 데이터.
    *   **시스템 자원 (System Assets):** AI 모델, API 키, 초기 설정값 등 시스템 구동에 필요한 초기 자본.
    *   **목표 정의 (Goal Definition):** 달성하고자 하는 최종 수익 목표(예: 월 $10k)를 명확히 수치화한 값.

#### B. Process (처리 단계): AI 기반의 Profit Path 설계 및 자동화
**(화면: 'Process' 박스 강조. Input이 복잡하게 얽혀서 정제되는 과정을 시각적으로 표현)**

**내레이션:** "두 번째 단계는 **Process**, 즉 이 투입된 자원을 수익으로 전환하는 핵심 알고리즘입니다. 여기서 AI의 힘을 빌려 데이터의 노이즈를 제거하고, 가장 효율적인 경로를 설계합니다."

*   **핵심 로직 (Logic Flow):**
    *   **데이터 정제 (Data Cleansing):** Input 데이터를 분석하여 불필요한 변수를 제거하고 핵심 신호만 추출하는 단계.
    *   **경로 최적화 (Path Optimization):** 정의된 목표에 따라 가장 높은 전환율을 보장하는 수익 경로(Profit Path)를 수학적으로 계산하고 설계하는 알고리즘.
    *   **자동화 모듈 연결 (Automation Module Linking):** Input 데이터를 기반으로 실제 실행 가능한 API 호출 및 자동화 스크립트를 연결하는 단계.

#### C. Output (산출 단계): 실행 가능한 수익 엔진 구축
**(화면: 'Output' 박스 강조. 깔끔하고 강력한 최종 결과물이 도출되는 모습)**

**내레이션:** "마지막 세 번째 단계는 **Output**, 즉 실제 돈이 흐르는 실행 가능한 시스템을 구축하는 것입니다. 이 단계에서 우리는 추상적인 계획을 구체적인 코드로 전환합니다."

*   **핵심 산출물 (Deliverables):**
    *   **Profit Path 시각화 맵:** Input에서 Output까지의 모든 경로와 각 단계별 수익 기여도를 오렌지색(`FF8C00`)으로 명확히 표시한 최종 지도.
    *   **자동화 스크립트 (Automation Scripts):** Process 단계를 자동화하는 실제 실행 가능한 Python/API 코드 모듈.
    *   **수익 경로 보고서 (Profit Path Report):** 시스템의 현재 상태와 잠재적 수익을 예측하는 데이터 기반 리포트.
