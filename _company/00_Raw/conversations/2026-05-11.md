### 1. 전체 레이아웃 및 구도 (Layout & Composition)

*   **스타일:** 다크 모드 기반의 테크니컬/미스터리한 분위기. 배경은 짙은 차콜 그레이 또는 블랙을 사용하여 오렌지색(`FF8C00`)이 극대화되도록 대비를 설정합니다.
*   **구도:** 중앙 집중형(Centralized) 흐름 구조를 채택하여 시청자의 시선이 자연스럽게 시스템의 시작점(Input)에서 최종 결과물(Output)로 이동하도록 유도합니다.
*   **레이아웃:** 수평적 흐름을 기본으로 하되, 핵심 모듈 간의 연결은 에너지 흐름처럼 역동적인 곡선과 선으로 표현하여 정적인 다이어그램이 아닌 '작동 시뮬레이션' 느낌을 부여합니다.

### 3. 애니메이션 흐름 (Animation Flow) - 역동성 극대화

Motion Graphic의 핵심은 정적인 흐름이 아닌 '에너지의 이동'을 표현하는 것입니다.

1.  **시작 (M1):** 화면 중앙에서 데이터(Input)가 빛처럼 생성되며 M1 노드로 진입합니다.
2.  **처리 (M2 - 핵심):** M1에서 들어온 에너지가 **오렌지색(`FF8C00`)의 에너지 흐름선**을 따라 M2 엔진 내부로 폭발적으로 유입됩니다. 이 과정에서 복잡한 데이터 처리(알고리즘)가 빠르게 시뮬레이션되며, 흐름선 자체가 동적으로 변화하고 빛나는 효과를 줍니다. (이 부분이 '시스템 해부'의 핵심 시각화)
3.  **변환 (M2 내부):** M2 내에서 여러 하위 모듈(Sub-modules)이 병렬로 작동하는 것을 보여주며, 각 모듈 간의 연결은 에너지 흐름으로 표현됩니다.
4.  **결과 도출 (M3):** M2 엔진을 통과한 최종 결과물(Output)이 강력한 오렌지색 빛을 발하며 M3 노드로 분출됩니다. 이 과정에서 수익 경로(`Profit Path`)가 시각적으로 연결되어, 시스템 작동이 곧 수익 창출로 이어짐을 명확히 보여줍니다.
5.  **마무리:** 최종 결과물(M3)은 화면 중앙에 고정되며, 오렌지색 강조 원칙이 전체 화면의 권위(Authority)를 완성합니다.

### 4. 색상 및 타이포그래피 적용 (Color & Typography Application)

*   **강조 색상:** **`#FF8C00` (Orange)**: 모든 핵심 흐름선, 엔진 내부 활성화 상태, 최종 결과물 박스에 적용하여 시각적 권위와 에너지의 흐름을 표현합니다.
*   **보조 색상:** 짙은 회색/블랙(배경), 밝은 흰색/밝은 회색(텍스트 및 기본 모듈 배경)을 사용하여 오렌지색이 압도적으로 돋보이게 합니다.

import json
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Dict, Any

# --- 1. JSON 구조 정의 (Designer 확정 기반) ---
class InputData(BaseModel):
    input_data: Dict[str, Any]
    system_config: Dict[str, Any]

class ProcessingResult(BaseModel):
    status: str
    processed_data: Dict[str, Any]
    profit_path: float
    system_metrics: Dict[str, float]

# --- 2. 핵심 로직 프로토타입 (Processing Engine) ---
def calculate_profit_path(input_data: Dict[str, Any], config: Dict[str, Any]) -> float:
    """
    Input 데이터와 시스템 설정을 기반으로 수익 경로를 계산하는 핵심 로직.
    실제 비즈니스 로직을 시뮬레이션합니다.
    """
    try:
        base_value = input_data.get("initial_value", 100)
        multiplier = config.get("profit_multiplier", 1.5)
        cost_factor = input_data.get("cost", 50)

        # 시뮬레이션 로직: (입력 * 승수) - 비용 조정
        profit = (base_value * multiplier) - cost_factor
        return round(profit, 2)
    except Exception as e:
        raise ValueError(f"Profit calculation failed: {e}")

def run_system_simulation(input_data: Dict[str, Any], system_config: Dict[str, Any]) -> ProcessingResult:
    """
    전체 시스템 작동 시뮬레이션을 실행하고 결과를 반환합니다.
    """
    try:
        profit = calculate_profit_path(input_data, system_config)

        # 시스템 메트릭 계산 (예시)
        metrics = {
            "processing_time_ms": 150,  # 시뮬레이션 시간
            "efficiency_score": round(system_config.get("efficiency", 0.8) * 100, 2),
            "risk_level": "Medium" if profit > 100 else "Low"
        }

        return ProcessingResult(
            status="Success",
            processed_data=input_data,
            profit_path=profit,
            system_metrics=metrics
        )
    except ValueError as ve:
        raise HTTPException(status_code=400, detail=str(ve))
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"Internal Simulation Error: {e}")

# --- 3. FastAPI 애플리케이션 설정 (API Endpoint) ---
app = FastAPI(title="System Simulator API")

@app.post("/simulate", response_model=ProcessingResult)
def simulate_system(data: InputData):
    """
    Input 데이터와 시스템 설정을 기반으로 수익 경로를 시뮬레이션합니다.
    """
    # JSON 구조 매핑 및 검증 (Pydantic이 이미 처리함)
    input_dict = data.model_dump()
    config_dict = data.system_config

    try:
        result = run_system_simulation(input_dict, config_dict)
        return result
    except HTTPException as e:
        raise e
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"Simulation failed: {e}")

@app.get("/config")
def get_config():
    """
    시스템 설정 정보를 제공합니다.
    """
    # 실제 환경에서는 환경변수나 DB에서 로드해야 하지만, 프로토타입이므로 하드코딩된 설정을 반환합니다.
    return {
